# Historical documentation

These documents record design-time intent, investigations, and earlier product
states. They are retained for rationale and archaeology, not as implementation
instructions.

They may contain old names (`ipod-sync`), old paths, superseded protocol
versions, incomplete plans, and assumptions that no longer hold. Use the
current documents in the parent directory and the code itself for present-day
behavior.

- `original-spec.md` — initial Windows-first product specification.
- `scsi-research.md` — original SCSI and SysInfoExtended investigation.
- `ipc-protocol-history.md` — append-only protocol history replaced by the
  current normative protocol reference.
- `LEARNINGS-history.md` — chronological incident and gate diary replaced by
  the concise root `LEARNINGS.md`.
- `design/` — selected implemented design records whose rationale remains
  useful.

Completed implementation plans and obsolete review reports are available from
Git history and are intentionally not retained in the working tree.
